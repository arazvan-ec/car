1 · Overview

App de análisis y comparativa de vehículos. Tras la unificación reciente, todo el dominio se modela como PipelineRun: cada análisis o comparativa es un run con pasos atómicos y fuentes versionadas. El frontend (React/Vite) consume la misma API REST que el servidor MCP, lo que permite tanto UI como agentes IA generar/leer análisis.

Endpoints

POST/api/v1/pipelines/runs/
Crear pipeline run (vehicle_analysis / vehicle_comparison). Devuelve detalle con steps inicializados según VEHICLE_STEP_TYPES o lista custom.
GET/api/v1/pipelines/runs/?pipeline_type=…&status_filter=…
Listar runs, filtrable por tipo/subject/estado.
GET/api/v1/pipelines/runs/{run_id}
Detalle: steps + data_sources con current_cache embebido.
POST/api/v1/pipelines/runs/{run_id}/rerun
Marca todos los pasos como stale (sin borrar sources).
PATCH/api/v1/pipelines/runs/{run_id}/steps/{step_type}
Actualiza status / structured_result / error_message.
POST/api/v1/pipelines/runs/{run_id}/steps/{step_type}/rerun
Marca el step como stale; propaga stale a steps que dependen de él.
POST/api/v1/pipelines/runs/{run_id}/steps/{step_type}/sources/
Añadir DataSource con CacheEntry. Si source_name ya existía, crea nueva versión, marca anterior is_current=false.
GET/api/v1/pipelines/runs/{run_id}/steps/{step_type}/sources/{source_name}/history
Historial completo de versiones de caché de una fuente.
GET/health
Health check (sin auth).
GET/docs
Swagger UI (sin auth).

7 · Known issues & tech debt

HIGH ISSUE-2 · CORS abierto con credenciales

api/app/main.py: allow_origins=["*"] + allow_credentials=True. Restringir a https://front-production-3a3e.up.railway.app y dominios de desarrollo.

HIGH ISSUE-3 · API key con default inseguro

api/app/core/config.py define API_KEY="change-me-in-production". Si la env var falta en Railway, la API queda abierta. Cambiar a Field sin default y dejar que Pydantic falle al arrancar.

MED ISSUE-4 · API key expuesta en bundle frontend

VITE_API_KEY se inyecta build-time y queda visible en el JS público. Migrar a auth con tokens de sesión emitidos por la API, o aceptar que la key del frontend es realmente un identificador, no un secreto.

MED ISSUE-5 · Sin rate limiting ni input validation estricta

AddSourceRequest acepta raw_content sin max_length. UpdateStepRequest.structured_result es Optional[dict] arbitrario. Añadir constraints Pydantic y, opcionalmente, slowapi para throttling por API key.

LOW ISSUE-7 · Sin tests automatizados

No hay carpeta tests/ ni en backend ni en frontend. Cualquier feature nueva debería incluir pytest (API) y/o Vitest (frontend).
